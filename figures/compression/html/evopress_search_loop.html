Figure. EvoPress search loop. Top row: starting from a parent configuration x(t) encoding per-layer sparsity levels, λ offspring are generated by level-switch mutations — each mutation shifts sparsity from one layer to another, preserving the global compression budget. A multi-step selection procedure evaluates candidates using increasing calibration token budgets T1 < T2 < ⋯ < TS, eliminating weaker offspring at each step; the parent is re-introduced at the final round (elitism). The fittest surviving candidate replaces the parent, and the loop repeats. Bottom row: fitness is the negative KL divergence between the pruned model and the full reference model; mutation count is biased toward single switches for stable search. The algorithm converges in 5–6 GPU hours and is provably optimal in O(k(n−k)/λ) generations for linear fitness functions.
