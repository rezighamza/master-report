Figure. DarwinLM two-phase structured pruning. Top row (Phase 1 — SLD pre-computation): for each transformer layer, an OBS-based structured pruning procedure is applied at Nl discrete sparsity levels, removing an increasing number of attention heads or MLP intermediate rows ranked by second-order saliency ΔL = w2i/2[H−1]ii. All resulting pruned weight matrices are stored in the Sparsity Level Database (SLD) — a lookup table indexed by (layer, module, level). Bottom row (Phase 2 — evolutionary search): starting from a parent configuration retrieved from the SLD, λ offspring are generated by level-switch mutations that swap sparsity levels between two layers within the same module group (Attn↔Attn or MLP↔MLP), preserving the global parameter budget. Each candidate is evaluated by a three-step training-aware selection procedure: offspring are fine-tuned on 10K, 50K, and 200K tokens progressively, with KL-divergence fitness measured against the full reference model after each fine-tuning stage. The best offspring replaces the parent, and the loop repeats until convergence.
