Figure. ZipLM combines layer-wise second-order structured pruning with runtime-aware search. Top row (global phase): given a pretrained model θ, calibration data 𝒞, and a target device with speedup ×S, candidate attention-head and FFN configurations are benchmarked on device to populate a per-layer runtime lookup table. A greedy search over that table then identifies the per-layer sparsity profile whose cumulative latency satisfies the budget ≤T, yielding a family of compressed models at discrete speedup targets. Bottom row (local phase, zoomed): for each layer ℓ selected by the profile, structured candidates (head or FFN group) are scored via the inverse-Hessian sub-block H−1qq; the lowest-scoring unit is removed, remaining weights are corrected by the second-order compensation ΔW, and H−1 is updated via a rank-1 outer-product step before the next removal is considered.
