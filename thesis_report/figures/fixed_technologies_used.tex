\section{Technical Architecture of the Solution}

The proposed solution follows a modular technical architecture. The implementation is organised around model loading, dataset preparation, pruning search, candidate evaluation, recovery fine-tuning, and final testing. The pretrained Transformer model is first loaded through the Hugging Face Transformers library. The dataset is then prepared and tokenized before being passed to the pruning procedure. During pruning, structured masks are applied to attention heads and feed-forward neurons in order to reduce the computational cost of the model while preserving as much task performance as possible. After the pruning step, the selected architecture is recovered through fine-tuning and evaluated using the target task metrics.

\subsection{Transformers}

Transformers is an open-source library developed by Hugging Face that provides a unified interface for using pretrained Transformer-based models. It includes implementations of widely used architectures such as BERT, RoBERTa, GPT, T5, and other models used in natural language processing. The library simplifies the process of loading pretrained checkpoints, tokenizers, configuration files, and task-specific model heads \cite{wolf2020transformers,huggingfaceTransformersDocs}.

In this work, the Transformers library was used to load the pretrained BERT model and interact with its internal modules. More specifically, it provided access to the encoder layers, attention heads, feed-forward submodules, configuration parameters, and model outputs required by the pruning procedure. This made it possible to apply structured masks, inspect the architecture, modify selected components, and fine-tune the pruned model within a standard implementation of BERT.
